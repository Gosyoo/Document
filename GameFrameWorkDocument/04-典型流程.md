# 第四章 · 典型流程示例

> 本章用时序图 + 流程图展示几个最常见的运行时流程，把前面的模块串起来。

## 1. 框架启动到模块第一次 Update

```mermaid
sequenceDiagram
  participant Main as Unity 主循环
  participant Entry as GameFrameworkEntry
  participant Mod as XxxManager(继承 GameFrameworkModule)

  Main->>Entry: GetModule<IEventManager>()
  Entry->>Entry: 反射获取 EventManager 类型
  Entry->>Mod: Activator.CreateInstance(EventManager)
  Entry->>Entry: 按 Priority 降序插入 LinkedList
  Entry-->>Main: 返回 IEventManager 引用

  loop 每帧 Update
    Main->>Entry: Update(elapseSeconds, realElapseSeconds)
    Entry->>Mod: foreach module.Update(...)
    Note over Entry,Mod: Priority 高的先 Update
  end

  Main->>Entry: Shutdown()
  Entry->>Mod: 反向遍历 module.Shutdown()
  Entry->>Entry: ReferencePool.ClearAll()
```

## 2. 完整启动流程（基于 Procedure）

```mermaid
flowchart TD
  P0[ProcedureLaunch<br/>读取 AppConfig / 注册事件]
  P1[ProcedureSplash<br/>启动画面]
  P2[ProcedureCheckVersion<br/>请求最新版本号]
  P3{需要更新?}
  P4[ProcedureUpdateResources<br/>下载差异]
  P5[ProcedureVerifyResources<br/>CRC32 校验]
  P6[ProcedurePreload<br/>预加载关键资源/数据表/本地化]
  P7[ProcedureLogin<br/>登录服务器]
  P8[ProcedureMain<br/>进入主城/主菜单]
  P0-->P1-->P2-->P3
  P3 -- 是 --> P4 --> P5 --> P6
  P3 -- 否 --> P6
  P6-->P7-->P8
```

实际项目中每个 Procedure 内部，会用 `ChangeState<NextState>(fsm)` 做状态切换。

## 3. 资源加载完整时序（LoadAsset）

```mermaid
sequenceDiagram
  participant Biz as 业务代码
  participant RM as ResourceManager
  participant RL as ResourceLoader
  participant TP as TaskPool<LoadResourceTaskBase>
  participant Agent as LoadResourceAgent
  participant Helper as ILoadResourceAgentHelper

  Biz->>RM: LoadAsset(name, callbacks, userData)
  RM->>RL: 解析 AssetInfo / ResourceInfo
  alt Asset 已在内存
    RL-->>Biz: 直接成功回调
  else 未加载
    RL->>TP: AddTask(new LoadAssetTask)
    TP->>Agent: Start(task) → CanResume
    Agent->>Helper: ReadFile(filePath)（或 ParseBytes）
    Helper-->>Agent: 完成
    Agent->>Helper: LoadAsset(resource, assetName, type)
    Helper-->>Agent: 资源就绪
    Agent->>RL: Done = true
    RL-->>Biz: LoadAssetSuccessCallback(asset)
  end
```

依赖资源：`LoadAssetTask` 会检测依赖，自动派发 `LoadDependencyAssetTask`，所有依赖都完成后才回调成功。

## 4. 资源更新流程（Updatable 模式）

```mermaid
sequenceDiagram
  participant App
  participant RM as ResourceManager
  participant VLP as VersionListProcessor
  participant Chk as ResourceChecker
  participant Up as ResourceUpdater
  participant Ver as ResourceVerifier
  participant DL as DownloadManager

  App->>RM: InitResources()
  RM->>VLP: UpdateVersionList(remoteUrl)
  VLP->>DL: 下载 GameFrameworkVersion.dat
  DL-->>VLP: 完成
  VLP-->>RM: VersionListReady
  RM->>Chk: CheckResources()
  Chk->>Chk: 对比 LocalVersionList vs UpdatableVersionList
  Chk-->>RM: List<ResourceNeedUpdate>
  RM->>Up: UpdateResources()
  loop 每个差异资源
    Up->>DL: AddDownload(resource)
    DL-->>Up: DownloadSuccess
  end
  Up-->>RM: AllUpdateComplete
  RM->>Ver: VerifyResources()
  Ver-->>RM: VerifyComplete
  RM-->>App: 资源就绪 → 进入下一 Procedure
```

## 5. UI 打开 / 关闭

### 打开

```mermaid
sequenceDiagram
  participant Biz
  participant UIM as UIManager
  participant Pool as ObjectPool<UIFormInstanceObject>
  participant Res as ResourceManager
  participant Helper as IUIFormHelper

  Biz->>UIM: OpenUIForm(assetName, group, priority, ...)
  UIM->>Pool: Spawn(assetName)
  alt 池中存在
    Pool-->>UIM: 直接复用 UIForm
  else 池中不存在
    UIM->>Res: LoadAsset(assetName, callbacks)
    Res-->>UIM: asset (prefab)
    UIM->>Helper: InstantiateUIForm(asset)
    Helper-->>UIM: gameObject
    UIM->>Pool: Register(包装为 UIFormInstanceObject)
  end
  UIM->>UIM: AddToUIGroup → 重排深度(SortByDepth)
  UIM-->>Biz: OpenUIFormSuccessEventArgs
  UIM->>Helper: OnInit / OnOpen 钩子调用
```

### 关闭

```mermaid
sequenceDiagram
  participant Biz
  participant UIM as UIManager
  participant Pool as ObjectPool

  Biz->>UIM: CloseUIForm(serialId)
  UIM->>UIM: 触发 OnClose 钩子
  UIM->>Pool: Unspawn(uiForm.Handle)
  Pool->>Pool: LastUseTime=Now，等待自动释放
```

## 6. 网络通信收发流程

```mermaid
sequenceDiagram
  participant Biz
  participant Ch as NetworkChannel
  participant Helper as INetworkChannelHelper
  participant Sock as Socket
  participant Handler as IPacketHandler

  Note over Biz: 发送
  Biz->>Ch: Send(packet)
  Ch->>Helper: Serialize(packet, sendStream)
  Ch->>Sock: BeginSend(buffer)

  Note over Biz: 心跳
  Ch-->>Helper: SendHeartBeat() 周期触发

  Note over Biz: 接收
  Sock->>Ch: BeginReceive 完成
  Ch->>Helper: DeserializePacketHeader(header)
  Ch->>Helper: DeserializePacket(header, body)
  Helper-->>Ch: Packet 实例
  Ch->>Handler: Handle(sender, packet) (按 Packet.Id 分发)
  Handler->>Biz: 业务回调
```

## 7. 事件系统的"线程安全"是怎么做到的？

```mermaid
sequenceDiagram
  participant T1 as 工作线程
  participant T2 as 主线程
  participant Pool as EventPool
  participant Q as 事件队列(锁保护)

  T1->>Pool: Fire(sender, args)
  Pool->>Q: lock(m_Events) { m_Events.Enqueue }

  loop 每帧
    T2->>Pool: Update(elapseSeconds, realElapseSeconds)
    Pool->>Q: lock { 一次性 dequeue 全部 }
    loop 每个事件
      Pool->>T2: handler(sender, args)
      Pool->>Pool: ReferencePool.Release(args)
    end
  end
```

> 💡 因此你在异步回调（HTTP/Socket 线程）里 `Fire` 完全安全，且事件处理逻辑保证在主线程执行。

## 8. 实体显示流程

```mermaid
sequenceDiagram
  participant Biz
  participant EM as EntityManager
  participant EG as EntityGroup
  participant Pool as ObjectPool
  participant Res as ResourceManager
  participant Helper as IEntityHelper

  Biz->>EM: ShowEntity(id, type, prefab, group, ud)
  EM->>EG: 找到 EntityGroup
  EG->>Pool: Spawn(prefabName)
  alt 池中有
    Pool-->>EG: EntityInstanceObject
  else 池中无
    EG->>Res: LoadAsset(prefab)
    Res-->>EG: asset
    EG->>Helper: InstantiateEntity(asset)
    Helper-->>EG: gameObject
    EG->>Pool: Register
  end
  EG->>EM: 实例就绪
  EM->>EM: CreateEntity(IEntity 接口实现)
  EM->>Helper: OnShow → OnRecycle 等钩子
  EM-->>Biz: ShowEntitySuccessEventArgs
```
