# 第五章 · 学习路径与实战指南

> 这一章给你一个**循序渐进的五阶段学习计划**，附实战练习。

## 🟢 第一阶段：理解骨架（约 2~3 小时）

### 目标
弄懂"模块如何被注册和驱动"。

### 推荐阅读顺序
1. **`Base/GameFrameworkEntry.cs`** —— 看 `GetModule` 与 `CreateModule` 是怎么把 Manager 反射出来的。
2. **`Base/GameFrameworkModule.cs`** —— 知道 `Update` / `Shutdown` / `Priority` 三件套。
3. **`Base/ReferencePool/`** 整个目录（4 个文件）—— 弄懂"为什么处处都要 IReference"。
4. **`Base/EventPool/`** 整个目录 —— 看清线程安全分发的实现。

### 课后小练
- ❓ 我自己写一个 `MyModule : GameFrameworkModule`，能不能被 `GameFrameworkEntry` 创建？
  > 不能直接用 `GetModule<T>()`（因为 `GetModule` 限定接口必须以 `GameFramework.` 开头）。这意味着框架想保持"内核闭合"。
- ❓ `ReferencePool.Acquire<T>()` 与 `new T()` 在多次调用下，GC 差异有多大？写个小 Demo 试试。

## 🟡 第二阶段：理解三大支柱（约 3~5 小时）

### 目标
吃透 FSM、Procedure、ObjectPool。

### 推荐阅读
1. **`Fsm/FsmState.cs`** + `Fsm/Fsm.cs` + `Fsm/FsmManager.cs`
2. **`Procedure/ProcedureBase.cs`** —— 你会发现它就是 `FsmState<IProcedureManager>` 的别名
3. **`ObjectPool/ObjectBase.cs`** + `ObjectPool/ObjectPoolManager.ObjectPool.cs`
4. **`ObjectPool/ObjectPoolManager.cs`** —— 创建/销毁对象池、各种泛型重载

### 课后小练
- 🛠 实现一个怪物 AI 状态机：Idle ↔ Chase ↔ Attack。
- 🛠 给一个 `IObjectPool<MyObj>` 设置 `AutoReleaseInterval=10`、`ExpireTime=30`，观察打日志。

## 🔵 第三阶段：资源管线（约 5~8 小时，难点）

### 目标
能讲清楚一份资源从远端到内存的完整链路。

### 推荐阅读顺序
1. **`Base/TaskPool/`** 整个目录（6 个文件）—— 任务/代理调度模型
2. **`Resource/IResourceManager.cs`** —— 全部能力概览（33KB，重点看 API 签名）
3. **`Resource/ResourceMode.cs`** —— 三种资源模式
4. 子系统按以下顺序串：
   ```
   PackageVersionList / UpdatableVersionList / LocalVersionList
     → ResourceManager.VersionListProcessor
     → ResourceManager.ResourceChecker
     → ResourceManager.ResourceUpdater
     → ResourceManager.ResourceVerifier
     → ResourceManager.ResourceLoader (含 LoadAssetTask / LoadResourceAgent)
   ```
5. **`FileSystem/FileSystem.cs`** —— 看自定义文件格式（HeaderData/StringData/BlockData）

### 课后小练
- 🛠 拿 StarForce 工程，改 `ResourceMode = Updatable`，观察启动时哪些 Procedure 被触发。
- ❓ `LoadAssetTask` 和 `LoadDependencyAssetTask` 的关系是什么？

## 🟣 第四阶段：业务模块举一反三（按需）

### 这些模块"套路相同"
```
ObjectPool ←─ Entity / UI / Sound 都用它
Resource   ←─ Entity / UI / Sound / Scene / Localization / DataTable / Config 都用它
TaskPool   ←─ Resource / Download / WebRequest / Network 都用它
```

> ✨ **看懂一个模块就等于看懂一类模块**。

### 推荐顺序
1. **UI** 完整看一遍（OpenUIForm 走一遍）
2. **Entity** —— 你会发现和 UI 套路 80% 一样
3. **Sound** —— 又是同一个套路 + 多了 `PlaySoundParams`
4. **Scene** —— 最简单的"包装层"模块
5. **Network** —— 看 `PacketHeader` → `Packet` → `IPacketHandler` 的处理链
6. **Download / WebRequest** —— 基本就是 TaskPool 的标准应用
7. **Config / DataTable / Localization** —— 注意它们都基于 `Base/DataProvider/`
8. **DataNode / Setting** —— 简单的"运行时变量"管理

### 课后小练
- 🛠 自己写一个 `SimpleHpBarUIForm`，让它从对象池打开 100 次，观察 OnOpen/OnClose/OnReveal/OnCover 哪些钩子被调用。
- 🛠 写一个 `DataTableHero`，从一份 CSV 加载 100 条配置。

## 🎯 学习心法（重要总结）

GameFramework 的所有模块几乎都遵循同一组"约定"：

| 约定 | 体现 |
|---|---|
| ① 接口对外、实现内部 sealed | `IXxxManager` / `internal sealed class XxxManager` |
| ② 引擎相关下沉到 `IXxxHelper` | `IEntityHelper` / `IUIFormHelper` / `ILoadResourceAgentHelper` |
| ③ 异步操作走 TaskPool / EventPool | Resource / Download / WebRequest / Network 都用任务池 |
| ④ 高频对象走 ReferencePool / ObjectPool | EventArgs / TaskBase / EntityInstanceObject 都实现 IReference |
| ⑤ 流程类逻辑走 Fsm / Procedure | 游戏主流程、AI、状态UI 都用 FSM |
